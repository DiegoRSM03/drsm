## Git Workflow

- **Branches**: `main` (prod) / `dev` (integration). **NEVER commit directly to either.**
- Use `feature/name` or `fix/name` branches. Squash merge via PR.
- **Conventional commits**: `feat:`, `fix:`, `chore:`, `refactor:`
- **Clean before committing** ŌĆö remove temporary files (screenshots, debug logs, test images) before staging. Never commit artifacts that aren't part of the deliverable.
- **No AI attribution** ŌĆö never add co-authors, AI mentions, or "generated by" in commits, PRs, code, or comments

## Code Quality Standards

### TypeScript

- No `any` ŌĆö use `unknown` + type guards
- Prefer `interface` over `type` (except one-liner unions)
- Extract props into named interfaces ŌĆö no inline prop types

### React

- **Server Components by default** ŌĆö only add `'use client'` when justified
- **No unnecessary `useEffect`** ŌĆö derive state during render, use event handlers for side effects, `useSyncExternalStore` for subscriptions. `useEffect` is a last resort for true synchronization with external systems.
- **Colocate state** ŌĆö keep state as close to where it's used as possible
- Memoize expensive computations with `useMemo`, not re-renders with `useEffect` + `setState`
- Use `useRef` for values that don't need re-renders (timers, DOM refs, previous values)

### Composition

- Prefer composition over props: `children`, compound components, named slots, render props, specialization
- Don't force it ŌĆö but always evaluate if a component can be decomposed or made more flexible through composition before reaching for prop flags or conditionals

### Styling

- **Mobile-first always** ŌĆö base styles are mobile, scale up with `sm:`, `md:`, `lg:`
- Use semantic color tokens (`text-foreground`, `bg-background`, `border-border`) ŌĆö never hardcode colors without theme-aware alternatives
- Derive patterns from existing components when unsure ŌĆö read the codebase first

### Accessibility (WCAG 2.2 AA)

- Every component must be accessible ŌĆö semantic HTML, keyboard navigable, screen-reader friendly
- All interactive elements need visible focus states and `aria-*` labels where appropriate
- Always respect `prefers-reduced-motion`

### Performance

- LCP < 2.5s, CLS < 0.1 ŌĆö measure, don't guess
- Lazy load below-fold content
- Images via `next/image` only ŌĆö no external optimization services
- Animations: transform and opacity only when possible. Always respect `prefers-reduced-motion`.

## Testing

**ALWAYS write unit tests** for every component created or modified. No exceptions.

- Jest + React Testing Library, colocated (`component.test.tsx`)
- Test behavior and accessibility, not implementation details
- Use `screen.getByRole` / `getByLabelText` over `getByTestId` when possible

## Design System

**Use the `drsm-design-system` skill** when building UI components. Full docs at `.claude/skills/drsm-design-system/SKILL.md`.

**Critical rule**: NO `rounded-*` classes on UI components (buttons, cards, inputs, tags). Sharp edges only. Circles allowed on decorative floating shapes only.

## Gotchas

- **No comments unless absolutely necessary** ŌĆö code should be self-explanatory
- **No "Open to Work" messaging** ŌĆö professional portfolio, not a job board
- **i18n keys must exist in both locales** (en, es) ŌĆö build will fail otherwise
- **No transparency on solid UI elements** ŌĆö cards, buttons, containers use solid colors. Transparency only on decorative/background elements.
- Components: default export + barrel file (`index.ts`) re-export. Follow existing patterns.

## Pre-Implementation Checklist

Before considering any work complete:

1. **Mobile-first** ŌĆö verify proper mobile, tablet, and desktop breakpoints
2. **Dark + light mode** ŌĆö every visual element correct in both themes
3. **Accessibility** ŌĆö semantic HTML, WCAG AA contrast, focus states, `aria-*` labels, reduced motion support
4. **Tests** ŌĆö unit tests written and passing
5. **Lint + build** ŌĆö `pnpm lint` and `pnpm build` pass
