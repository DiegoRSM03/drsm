## Git Workflow

- **Branches**: `main` (prod) / `dev` (integration). **NEVER commit directly to either.**
- Use `feature/name` or `fix/name` branches. Squash merge via PR.
- **Conventional commits**: `feat:`, `fix:`, `chore:`, `refactor:`
- **Clean before committing** ‚Äî remove temporary files (screenshots, debug logs, test images) before staging. Never commit artifacts that aren't part of the deliverable.
- **No AI attribution** ‚Äî never add co-authors, AI mentions, or "generated by" in commits, PRs, code, or comments

## Code Quality Standards

### TypeScript

- No `any` ‚Äî use `unknown` + type guards
- Prefer `interface` over `type` (except one-liner unions)
- Extract props into named interfaces ‚Äî no inline prop types

### React

- **Server Components by default** ‚Äî only add `'use client'` when justified
- **No unnecessary `useEffect`** ‚Äî derive state during render, use event handlers for side effects, `useSyncExternalStore` for subscriptions. `useEffect` is a last resort for true synchronization with external systems.
- **Colocate state** ‚Äî keep state as close to where it's used as possible
- Memoize expensive computations with `useMemo`, not re-renders with `useEffect` + `setState`
- Use `useRef` for values that don't need re-renders (timers, DOM refs, previous values)

### Composition

- Prefer composition over props: `children`, compound components, named slots, render props, specialization
- Don't force it ‚Äî but always evaluate if a component can be decomposed or made more flexible through composition before reaching for prop flags or conditionals

### Styling

- **Mobile-first always** ‚Äî base styles are mobile, scale up with `sm:`, `md:`, `lg:`
- Use semantic color tokens (`text-foreground`, `bg-background`, `border-border`) ‚Äî never hardcode colors without theme-aware alternatives
- Derive patterns from existing components when unsure ‚Äî read the codebase first

### Accessibility (WCAG 2.2 AA)

- Every component must be accessible ‚Äî semantic HTML, keyboard navigable, screen-reader friendly
- All interactive elements need visible focus states and `aria-*` labels where appropriate
- Always respect `prefers-reduced-motion`

### Performance

- LCP < 2.5s, CLS < 0.1 ‚Äî measure, don't guess
- Lazy load below-fold content
- Images via `next/image` only ‚Äî no external optimization services
- Animations: transform and opacity only when possible

## Testing

**ALWAYS write unit tests** for every component created or modified. No exceptions.

- Jest + React Testing Library, colocated (`component.test.tsx`)
- Test behavior and accessibility, not implementation details
- Use `screen.getByRole` / `getByLabelText` over `getByTestId` when possible

## Design System

**Use the `drsm-design-system` skill** when building UI components. Full docs at `.claude/skills/drsm-design-system/SKILL.md`.

**Critical rule**: NO `rounded-*` classes on UI components (buttons, cards, inputs, tags). Sharp edges only. Circles allowed on decorative floating shapes only.

## Gotchas

- **No comments unless absolutely necessary** ‚Äî code should be self-explanatory
- **No "Open to Work" messaging** ‚Äî professional portfolio, not a job board
- **i18n keys must exist in both locales** (en, es) ‚Äî build will fail otherwise
- **No transparency on solid UI elements** ‚Äî cards, buttons, containers use solid colors. Transparency only on decorative/background elements.
- Components: default export + barrel file (`index.ts`) re-export. Follow existing patterns.

## Spec Files

All spec files live in `/spec` at the project root. Create a new spec file before starting any feature or improvement work.

## Pre-Implementation Checklist

Before considering any work complete:

1. **Visual verification** ‚Äî every UI change must be verified across all 6 combinations: mobile/tablet/desktop √ó dark/light. Don't assume a fix for one viewport or theme works for the others. Hardcoded colors on themed surfaces break in the opposite mode. Spacing tweaks for mobile can overflow on tablet.
2. **Tests** ‚Äî unit tests written and passing
3. **Lint + build** ‚Äî `pnpm lint` and `pnpm build` pass
